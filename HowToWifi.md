# Tab5 WiFi Bringup — Complete How-To

> **Status:** Confirmed working as of 2026-05-24.
> Scan returns results; full TCP/HTTP stack available via standard Arduino WiFi API.

---

## Architecture Overview

The M5Stack Tab5 is a **dual-chip design**:

```
┌─────────────────────────────────────────────┐
│  ESP32-P4  (main processor)                 │
│  360 MHz RISC-V, 32 MB PSRAM, 16 MB flash   │
│  MIPI-DSI display, GT911 touch, I/O         │
│                                              │
│  ┌─────────────────────────┐                 │
│  │  ESP-Hosted HOST stack  │                 │
│  │  (bundled in platform)  │                 │
│  └──────────┬──────────────┘                 │
│             │  SDIO bus                      │
│        CLK=12  CMD=13                        │
│        D0=11   D1=10                         │
│        D2=9    D3=8    RST=15                │
└─────────────┼───────────────────────────────┘
              │
┌─────────────▼───────────────────────────────┐
│  ESP32-C6  (WiFi / BT co-processor)         │
│  WiFi 6 (802.11ax) + Bluetooth 5            │
│                                              │
│  ESP-Hosted SLAVE firmware v2.12.6          │
│  (factory-flashed, no reflash needed)        │
└─────────────────────────────────────────────┘
```

The P4 never drives a WiFi radio directly. Every WiFi operation (scan, connect,
send, receive) goes through the **ESP-Hosted** protocol over SDIO to the C6.
From application code this is completely transparent — the standard Arduino WiFi
API works as expected.

## Required Code Patches

Two bugs in the library layer must be fixed manually before WiFi (or even
`M5.begin()`) will work reliably. These are in `.pio/libdeps/` which is
regenerated by PlatformIO — re-apply after any `pio run` that wipes libdeps.

### Bug 1 — `tft` static initialisation order fiasco (`src/main.cpp`)

If you alias the display via a file-scope static reference, bind it to
`M5.Display` (a value member) rather than `M5.Lcd` (a reference member).

```cpp
// ❌ WRONG — crashes: M5.Lcd's internal pointer is NULL before M5Unified
//            constructor runs; if main.cpp statics init first, tft = nullptr
static M5GFX& tft = M5.Lcd;

// ✅ CORRECT — &M5.Display is a link-time constant, safe at any init order
static M5GFX& tft = M5.Display;
```

**Crash symptom:** `Load access fault` at `LGFXBase::setRotation` line 61,
`lw a0,8(a0)`, MEPC = 0x40003c26, A0 = 0x00000000.

### Bug 2 — M5GFX I²C bus corruption on IDF 5.5 (`.pio/libdeps/*/common.cpp`)

**File:** `.pio/libdeps/<env>/M5GFX/src/lgfx/v1/platforms/esp32/common.cpp`

**Root cause:** IDF 5.5 introduced a new I²C master driver
(`i2c_new_master_bus` / `i2c_del_master_bus`). M5GFX 0.2.21 calls `Wire.end()`
+ `Wire.begin()` multiple times during autodetect. On IDF 5.5, the del+create
cycle corrupts internal driver state; after ~5 iterations `bus_handle` becomes
NULL and the next I²C write crashes.

**Fix — part A:** In `release()` (around line 1319), wrap the hardware teardown
in an IDF version guard:

```cpp
// Replace the block inside `if (i2c_context[i2c_port].initialized) { ... }`
// with:

        i2c_context[i2c_port].initialized = false;
#if defined (ESP_IDF_VERSION_VAL) && (ESP_IDF_VERSION >= ESP_IDF_VERSION_VAL(5, 4, 0))
        // IDF 5.4+ NG driver: skip Wire.end() and pin reset.
        // Keep the bus alive; init() reuses it via the s_wire_init guard.
#else
#if defined ( ARDUINO ) && __has_include (<Wire.h>) && defined ( ESP_IDF_VERSION_VAL )
 #if ESP_IDF_VERSION >= ESP_IDF_VERSION_VAL(4, 0, 0)
  #if defined ARDUINO_ESP32_GIT_VER
    #if ARDUINO_ESP32_GIT_VER != 0x44c11981
      // ... existing twowire->end() code ...
    #endif
  #endif
 #endif
// ... existing #elif / #else branches ...
        if ((int)i2c_context[i2c_port].pin_scl >= 0) {
          pinMode(i2c_context[i2c_port].pin_scl, pin_mode_t::input_pullup);
        }
        if ((int)i2c_context[i2c_port].pin_sda >= 0) {
          pinMode(i2c_context[i2c_port].pin_sda, pin_mode_t::input_pullup);
        }
#endif  // !(IDF >= 5.4)
```

**Fix — part B:** In `init()` (around line 1442), call `Wire.begin()` only once
per port:

```cpp
// Replace:
 #if defined ( USE_TWOWIRE_SETPINS )
      twowire->begin();
 #else
      twowire->begin((int)pin_sda, (int)pin_scl);
 #endif

// With:
 #if defined (ESP_IDF_VERSION_VAL) && (ESP_IDF_VERSION >= ESP_IDF_VERSION_VAL(5, 4, 0))
      { static bool s_wire_init[I2C_NUM_MAX] = {};
        if (!s_wire_init[i2c_port]) {
          s_wire_init[i2c_port] = true;
  #if defined ( USE_TWOWIRE_SETPINS )
          twowire->begin();
  #else
          twowire->begin((int)pin_sda, (int)pin_scl);
  #endif
        }
      }
 #elif defined ( USE_TWOWIRE_SETPINS )
      twowire->begin();
 #else
      twowire->begin((int)pin_sda, (int)pin_scl);
 #endif
```

Apply to **all four** Tab5 libdeps copies (one per environment):
```
.pio/libdeps/firmware_m5tab5/M5GFX/src/lgfx/v1/platforms/esp32/common.cpp
.pio/libdeps/firmware_m5tab5_c6_scan/M5GFX/...
.pio/libdeps/firmware_m5tab5_c6_sdio/M5GFX/...
.pio/libdeps/firmware_m5tab5_wifi_direct/M5GFX/...
```

> **Persistence note:** PlatformIO regenerates `.pio/libdeps/` on `pio run` if
> library versions change. Keep a copy of these diffs (or the full patched files)
> in version control or in a `tools/patch_m5gfx.py` post-install script so they
> survive a clean rebuild.

## WiFi Initialisation Code

### SDIO pin constants

```cpp
// ESP32-C6 co-processor SDIO bus — verified on Tab5 hardware
#define TAB5_SDIO_CLK  12
#define TAB5_SDIO_CMD  13
#define TAB5_SDIO_D0   11
#define TAB5_SDIO_D1   10
#define TAB5_SDIO_D2    9
#define TAB5_SDIO_D3    8
#define TAB5_SDIO_RST  15
```

### Mandatory init sequence

```cpp
#include <WiFi.h>

void setup() {
    Serial.begin(115200);

    // ── Display / touch init ────────────────────────────────────────────
    auto cfg = M5.config();
    M5.begin(cfg);           // MUST come before WiFi — M5GFX owns I²C/GPIO
    tft.setRotation(1);      // cosmetic only; touch mapping done inside M5.begin()

    // ── WiFi init ───────────────────────────────────────────────────────
    // 1. Give the C6 time to finish booting its ESP-Hosted SDIO slave firmware.
    //    The factory firmware starts automatically; no reset pulse needed.
    delay(1500);

    // 2. Route the SDIO bus to the correct pins BEFORE WiFi.mode().
    //    WiFi.setPins() only exists in pioarduino 55.03.38-1 / Arduino 3.3.8+.
    //    Without this call the bus defaults to wrong pins → CMD5 OCR timeout.
    WiFi.setPins(TAB5_SDIO_CLK, TAB5_SDIO_CMD,
                 TAB5_SDIO_D0,  TAB5_SDIO_D1,
                 TAB5_SDIO_D2,  TAB5_SDIO_D3,
                 TAB5_SDIO_RST);

    // 3. Start the ESP-Hosted host stack (SDIO enumeration happens here).
    WiFi.mode(WIFI_STA);
}
```

### Scanning

```cpp
void wifiScan() {
    int n = WiFi.scanNetworks(/* async= */ false, /* show_hidden= */ true);
    if (n < 0) {
        Serial.printf("[wifi] scan failed: %d\n", n);
        return;
    }
    Serial.printf("[wifi] found %d networks\n", n);
    for (int i = 0; i < n; i++) {
        Serial.printf("  [%2d] RSSI %4d  Ch %2d  %-32s  %s\n",
            i,
            WiFi.RSSI(i),
            WiFi.channel(i),
            WiFi.SSID(i).c_str(),
            WiFi.encryptionType(i) == WIFI_AUTH_OPEN ? "open" : "secured");
    }
    WiFi.scanDelete();
}
```

### Connecting to an AP

```cpp
void wifiConnect(const char* ssid, const char* password) {
    Serial.printf("[wifi] connecting to %s …\n", ssid);
    WiFi.begin(ssid, password);

    uint32_t deadline = millis() + 15000;
    while (WiFi.status() != WL_CONNECTED && millis() < deadline) {
        delay(500);
        Serial.print('.');
    }
    Serial.println();

    if (WiFi.status() == WL_CONNECTED) {
        Serial.printf("[wifi] connected  IP=%s  RSSI=%d dBm\n",
            WiFi.localIP().toString().c_str(),
            WiFi.RSSI());
    } else {
        Serial.printf("[wifi] connection failed (status=%d)\n", WiFi.status());
    }
}
```

### Making an HTTP request

```cpp
#include <HTTPClient.h>

void httpGet(const char* url) {
    if (WiFi.status() != WL_CONNECTED) return;

    HTTPClient http;
    http.begin(url);
    int code = http.GET();
    if (code > 0) {
        Serial.printf("[http] %d  body=%d bytes\n", code, http.getSize());
        // String body = http.getString();  // grab response body if needed
    } else {
        Serial.printf("[http] error: %s\n", http.errorToString(code).c_str());
    }
    http.end();
}
```

## Flashing the Board

Tab5 uses USB-Serial/JTAG (USB-CDC). To enter download mode:

1. Hold the **RESET** button for ~2 seconds until the **internal green LED blinks rapidly**
2. Release RESET
3. Flash immediately:

```powershell
# Windows — bypass PlatformIO's display thread to avoid cp1252 Unicode errors
$env:PYTHONIOENCODING = "utf-8"
$env:PYTHONUTF8 = "1"
python "$env:USERPROFILE\.platformio\packages\tool-esptoolpy\esptool.py" `
    --chip esp32p4 --port COM4 --baud 921600 `
    --before default_reset --after hard_reset `
    write-flash -z --flash-mode keep --flash-freq keep --flash-size keep `
    0x0 ".pio\build\firmware_m5tab5_c6_scan\firmware.factory.bin"
```

Or via PlatformIO (works if the cp1252 issue does not affect your terminal):
```
pio run -e firmware_m5tab5_c6_scan -t upload --upload-port COM4
```

## Troubleshooting

### `send_op_cond (1) returned 0x107` then FreeRTOS abort

**Cause:** `WiFi.setPins()` was not called before `WiFi.mode()`, so the SDIO bus
defaulted to wrong pins. The C6 never responded to CMD5.

**Fix:** Add the `WiFi.setPins(...)` call exactly as shown above, before any
`WiFi.mode()` or `WiFi.begin()` call.

---

### `Load access fault` crash in `M5.begin()` — repeated `i2cInit` messages

**Cause:** M5GFX `common.cpp` patch (Bug 2 above) has not been applied.

Look for this pattern in serial output (5× `i2cInit` before crash):
```
[I][esp32-hal-i2c-ng.c:112] i2cInit(): Initializing I2C Master: num=1 ...
[I][esp32-hal-i2c-ng.c:112] i2cInit(): Initializing I2C Master: num=1 ...
[I][esp32-hal-i2c-ng.c:112] i2cInit(): Initializing I2C Master: num=1 ...
[I][esp32-hal-i2c-ng.c:112] i2cInit(): Initializing I2C Master: num=1 ...
[I][esp32-hal-i2c-ng.c:112] i2cInit(): Initializing I2C Master: num=1 ...
Guru Meditation Error: Core 1 panic'ed (Load access fault)
```

**Fix:** Apply M5GFX `release()` + `init()` patches from Bug 2 above.

---

### `Load access fault` in `tft.setRotation()` after `M5.begin()` succeeds

**Cause:** `static M5GFX& tft = M5.Lcd` static init order fiasco (Bug 1 above).

Signature: crash at `LGFXBase.cpp:61`, MEPC in M5GFX flash code, `A0=0x00000000`.

**Fix:** Change `M5.Lcd` → `M5.Display` in the `tft` reference declaration.

---

### WiFi connected but no TCP traffic / DHCP fails

Not observed on this hardware, but if it occurs: check that `WiFi.mode(WIFI_STA)`
returned `true` before calling `WiFi.begin()`. A `false` return means the
ESP-Hosted SDIO init failed silently — check serial for `ESP-Hosted initialized!`.

---

### Touch stops working after WiFi init

**Cause:** Something called `SPI.begin()` with default pins after `M5.begin()`,
and GPIO32 (GT911 I²C SCL) was claimed by the SPI peripheral.

**Fix:** Ensure no SD card or SPI init happens with `PIN_SD_*` = -1. SD is
disabled (`BOARD_HAS_SD = false`) for Tab5 until correct SPI3 pins are confirmed.
See CLAUDE.md → "SD card / GPIO32 collision".

---

### No output on serial monitor after flash + reset

The USB-CDC port disconnects briefly on hard reset. Start the monitor first, then
reset, or kill and restart the monitor after the device reconnects. The boot banner
appears within ~200 ms of USB re-enumeration.

## Key Facts to Remember

| Fact | Detail |
|---|---|
| C6 firmware | Factory-flashed ESP-Hosted SDIO slave v2.12.6 — **no reflash ever needed** |
| `WiFi.setPins()` | Must be called **before** `WiFi.mode()` — order is critical |
| Boot delay | `delay(1500)` before `WiFi.setPins()` — C6 needs time to boot |
| GT911 I²C pins | SDA=GPIO31, SCL=GPIO32 — never let SPI or anything else claim GPIO32 |
| `M5.begin()` owns everything | Never re-init touch, I²C expander, or GPIO32/31 after `M5.begin()` |
| M5GFX patches | In `.pio/libdeps/` — must be re-applied after libdeps regenerate |
| Platform | pioarduino only — `espressif32` official platform has unstable P4 SDIO WiFi |
| Flashing | Hold RESET ~2 s until green LED blinks, then release — do not hold through the blink |
